## Overview

`CatalogEntry` represents a commercial offering position - what the organization currently offers to customers. While `Product` defines what something IS (business/operational definition), `CatalogEntry` defines that something is FOR SALE (commercial availability).

**Package:** `com.softwarearchetypes.product`

**Source:** `/product/src/main/java/com/softwarearchetypes/product/CatalogEntry.java:22`

## Key Differences from Product

- **displayName**: Marketing name (vs technical name)
- **description**: Sales copy (vs technical description)
- **categories**: For navigation/search
- **validity**: When it's available for purchase
- **metadata**: Flexible attributes (featured, badges, promotions, etc.)

### builder


Creates a new builder for constructing CatalogEntry instances.

<ResponseField name="return" type="Builder">
  A new Builder instance
</ResponseField>

**Example:**
```java
CatalogEntry entry = CatalogEntry.builder()
    .id(CatalogEntryId.random())
    .displayName("Premium Laptop Bundle")
    .description("High-performance laptop with accessories")
    .product(myProduct)
    .category("Electronics")
    .category("Computers")
    .validity(Validity.from(LocalDate.now()))
    .withMetadata("featured", "true")
    .withMetadata("badge", "Best Seller")
    .build();
```
